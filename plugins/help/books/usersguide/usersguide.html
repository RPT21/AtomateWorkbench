User's Guide

The Workbench

The Workbench is an integrated environment for nanostructure synthesis. It provides a friendly graphical user interface (GUI) to control efficiently various hardware connected to the computer. It saves and manages process recipes, and it automatically acquires, stores, and manages data gathered during the process.

An important safety feature to the Workbench program is the Safety Interlock, accessed through the View menu or red or green circular button in the toolbar. The dialog that appears shows a list of devices associated with hardware that form part of the interlock. Clicking an item lets you see details and what type of error, if any, is occurring. It is unsafe to run the recipe if the button is red and the Workbench will not allow the system to run.

The Workbench shows the Edit Perspective window separated into the following panels: Grid Editor, Graph Editor, Extended Editor and Status Monitor.

Grid Editor

Each row of the Grid Editor represents a step in the recipe. Each cell in that row represents the setpoint of a device for that step, and, in particular, the duration cell indicates how long a step will last.

The arrow keys move the cursor between adjacent cells. Changing rows changes the current selected step. The <Enter> key and <Tab> key moves the cursor to the next cell.

Clicking on a cell will select that cell, and double clicking or hitting the space bar will allow editing of the cell.

Graph Editor

The graph editor shows a graphical representation of the recipe. It contains colored plots corresponding to each device's setpoints.

Extended Editor

The Extended Editor provides options for each step. It consists of "Step Properties" and device-specific options.

Step Properties are:

  • Duration which provides another place at which to enter the duration for a step.
  • Loop Step allows multiple loops with the following parameters:
    • Enclosing Steps are the number of steps to be included in the loop beginning with the step that is chosen.
    • Number of Loops is the number of times the looped steps are repeated.
      Example: To repeat a sequence of 3 steps 4 times, enter "3" for and "3" for Number of Loops.
  • Conditionals is a list of programmable conditional tests (if…then…) with which each step can be altered while the recipe is running. See below for more information.

Conditionals Editor

Conditionals are sets of if ... then ... statements whose evaluation, when True, results in an in-situ alteration of the recipe. Multiple sets of conditionals may be used in a single step.

Click the Edit button in the Extended Editor to open the window "Conditional Tests". To add new conditional tests , click Add. The window "Tests Group Editor" opens.

Each set of conditionals has a user-defined Action Group Name that appears in the conditionals field in the Step Properties of the Extended Editor. All statements within a set are joined with AND/OR by selecting All/Any of these must be true radio button, respectively. New statements are added by clicking the More button. Conditionals may be added, modified, or deleted.

Three fields appear: the first is the name of the heating device (heating zone / UP150 or Eurotherm) in the recipe, the second is a comparator ("<", ">", etc.), and the third is the value to which the device's output is compared. These statements are deleted by clicking the Fewer button. The three consequent actions of a test with a "true" result are selected with the following radio buttons:

  • Abort Recipe, which shuts down the current recipe and begins purging,
  • Hold Step, which extends the current step, and
  • Advance to next step, which shortens the current step.

Return to the recipe is forbidden without a name for the conditional set.

Device Editor

The device editor allows the user to manage the devices in the system. The heating element contains options such as ramp rate and the gas flow elements show the flow rate with use of the gas correction factor (GCF) once the hardware has been selected. (See Device section for more information).
